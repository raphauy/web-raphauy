# PRD — raphauy.dev

## 1. Resumen Ejecutivo

**Qué es:** Portfolio personal de Raphael, freelancer software engineer, mostrando 30+ proyectos de forma impactante.

**Para quién:** Potenciales clientes que evalúan contratar a Raphael para desarrollo de software.

**Objetivo principal:** Reemplazar el sitio actual (HTML estático con Bootstrap, desactualizado) por un portfolio moderno, rápido y visualmente atractivo que demuestre la amplitud y calidad del trabajo realizado.

**Dominios:** `raphauy.dev` + `rapha.uy` apuntando al mismo sitio.

### Schema de `data.json`

```jsonc
{
  // Identificación
  "slug": "qeeup",                          // string — URL slug, único
  "name": "QeeUp",                           // string — nombre para mostrar
  "tagline": "Plataforma multi-tenant de ticketing con chatbot IA", // string — una línea

  // Descripción
  "description": "...",                      // string — párrafo descriptivo (2-4 oraciones)
  "highlights": [                            // string[] — bullets de features clave
    "Chatbot IA con contexto de negocio",
    "Integración WhatsApp vía Evolution API",
    "Multi-tenant con roles y permisos"
  ],

  // Clasificación
  "type": "web-app",                         // "web-app" | "mobile-app" | "api" | "cli" | "library" | "landing" | "other"
  "tags": ["saas", "ai", "chatbot", "multi-tenant"], // string[] — tags libres para filtrar
  "tech": ["Next.js", "Convex", "Tailwind CSS", "OpenAI"], // string[] — tecnologías principales

  // Fechas
  "startDate": "2025-06",                   // string (YYYY-MM) — inicio del proyecto
  "endDate": null,                           // string | null — null si está activo

  // Stats (generadas por el comando de discovery)
  "stats": {
    "languages": {                           // Record<string, number> — líneas por lenguaje
      "TypeScript": 12400,
      "CSS": 800
    },
    "totalLines": 13200                      // number — total de líneas de código
  },

  // Links
  "url": "https://qeeup.com",               // string | null — sitio en producción
  "repoUrl": null,                           // string | null — repo público (si aplica)

  // Media
  "screenshots": [                           // string[] — nombres de archivo en screenshots/
    "hero.png",
    "dashboard.png",
    "chat.png"
  ],

  // IA
  "builtWithAI": true,                       // boolean — si se usaron agentes IA en el desarrollo
  "aiDetails": "Claude Code para scaffolding y features principales" // string | null — detalle del uso de IA
}
```

### Manejo de screenshots

- Se almacenan en `projects/<slug>/screenshots/`.
- La primera imagen del array `screenshots` se usa como hero/preview en la timeline.
- Formatos soportados: PNG, JPG, WebP.
- Next.js optimiza las imágenes con `next/image` en build time.
- Tamaño recomendado: 1280×720 mínimo, aspect ratio 16:9.

## 4. Comando de Discovery (Claude Code)

En vez de un script custom, el discovery de proyectos se hace con un **comando de Claude Code** (`/discover`). Claude Code ya tiene las herramientas para leer archivos, buscar patrones, ejecutar comandos y generar JSON estructurado.

### Invocación

Desde el directorio del sitio (`web-raphauy.dev`), ejecutar:

```
/discover /ruta/al/repo --slug mi-proyecto
```

### Qué debe hacer el comando

#### Paso 1 — Recopilar contexto del repo

- **package.json**: leer `name`, `dependencies`, `devDependencies` para inferir tech stack.
- **README.md / PRD.md / CLAUDE.md**: leer para entender el propósito, features y contexto del proyecto.
- **Contar líneas**: ejecutar `cloc --json .` en el repo para obtener líneas por lenguaje.
- **Detectar uso de IA**:
  - Buscar archivos: `CLAUDE.md`, `.cursor/`, `.github/copilot`, `.aider*`
  - Buscar en git log: `git log --oneline --all | grep -i "Co-Authored-By: Claude\|Generated by\|AI\|copilot\|cursor\|aider"` (primeros 500 commits)

#### Paso 2 — Generar datos estructurados

Con todo el contexto recopilado, generar los campos del `data.json`:

- `name`: nombre legible del proyecto (no el slug de npm).
- `tagline`: una oración que describe el proyecto (máx 80 chars).
- `description`: párrafo descriptivo de 2-4 oraciones orientado a un potencial cliente.
- `highlights`: 3-5 bullets de features/logros clave.
- `type`: clasificar en `"web-app" | "mobile-app" | "api" | "cli" | "library" | "landing" | "other"`.
- `tags`: 3-8 tags relevantes para filtrado (lowercase, sin duplicar tech).
- `tech`: tecnologías principales detectadas desde dependencies + código.
- `stats.languages`: del output de cloc (solo lenguajes con >50 líneas).
- `stats.totalLines`: suma total.
- `builtWithAI`: `true` si se encontraron indicadores de IA.
- `aiDetails`: descripción breve del uso de IA si `builtWithAI` es true.

#### Paso 3 — Escribir/actualizar `data.json`

- Si `projects/<slug>/data.json` ya existe: **merge** preservando campos manuales (`screenshots`, `url`, `repoUrl`, `startDate`, `endDate`). Actualizar solo los campos generados.
- Si no existe: crear `projects/<slug>/data.json` con todos los campos. Los campos manuales quedan con valores por defecto (`screenshots: []`, `url: null`, etc.).
- Crear el directorio `projects/<slug>/screenshots/` si no existe.

### Notas

- Se ejecuta manualmente en dev. El desarrollador revisa el JSON generado, ajusta lo necesario, y hace commit + push.
- Las screenshots se agregan manualmente a la carpeta correspondiente.
- `startDate` se puede inferir del primer commit del repo (`git log --reverse --format=%cs | head -1`) si no está ya definido.

## 5. Páginas y Componentes

### Página principal (`/`)

Layout vertical con tres secciones:

#### 5.1 Header / Hero compacto

- Nombre: "Raphael" + rol: "Freelance Software Engineer"
- Bio breve (2-3 líneas)
- Links de contacto: email, GitHub, LinkedIn
- Toggle dark/light mode
- Ocupa poco espacio vertical — el foco está en los proyectos.

#### 5.2 Project Activity Grid

Cuadrícula estilo GitHub contributions calendar ubicada entre el header y la timeline.

- **Rango:** últimos 12 meses.
- **Granularidad:** cada cuadradito = 1 semana.
- **Color:** coloreado si se creó un proyecto esa semana (basado en `startDate`). Intensidad del color según cantidad de proyectos en esa semana (si hay más de uno).
- **Hover tooltip:** muestra nombre del proyecto y fecha.
- **Responsive:** se comprime en mobile mostrando menos semanas.

#### 5.3 Timeline vertical con project cards

- Orden cronológico descendente (más reciente primero).
- Línea vertical central con puntos por proyecto (desktop). En mobile, línea a la izquierda.
- Cada card muestra:
  - Screenshot hero (primera imagen)
  - Nombre + tagline
  - Tech badges (las principales, max ~5)
  - AI Badge si `builtWithAI === true`
  - Fecha (mes/año)
- Click en card → navega a `/projects/[slug]`

#### 5.4 FilterBar

- Barra fija (sticky) encima de la timeline.
- **Filtro por tecnología:** multi-select con las tecnologías más frecuentes extraídas de todos los proyectos.
- Filtra la timeline en tiempo real (client-side).
- URL search params para compartir filtro (`?tech=Next.js,Convex`).

### Página de detalle (`/projects/[slug]`)

- **Hero:** screenshot principal a ancho completo con overlay del nombre.
- **Info:** description completa + highlights (bullets).
- **Galería:** grid de screenshots con lightbox al hacer click.
- **Stats sidebar:**
  - Lenguajes (barras proporcionales)
  - Total de líneas
  - Duración del proyecto (startDate → endDate o "En curso")
  - Links (sitio, repo)
- **Tech stack:** badges de todas las tecnologías.
- **AI Badge:** si aplica, badge prominente con `aiDetails`.
- **Navegación:** flechas prev/next entre proyectos + link "← Volver".

### Componentes

| Componente | Descripción |
|-----------|-------------|
| `TimelineCard` | Card de proyecto en la timeline. Props: project data. Animación scroll-reveal con Framer Motion. |
| `ProjectDetail` | Layout completo de la página de detalle. |
| `FilterBar` | Barra de filtros sticky con multi-select de tecnologías. |
| `TechBadge` | Badge con nombre de tecnología. Color por categoría (frontend, backend, db, etc.). |
| `AIBadge` | Badge "Built with AI Agents". Estilo distintivo (gradiente/glow). Tooltip con `aiDetails`. |
| `ProjectActivityGrid` | Cuadrícula de actividad estilo GitHub. |
| `ScreenshotGallery` | Grid de screenshots con lightbox. |
| `ThemeToggle` | Toggle dark/light mode. |
| `StatsBar` | Barras de lenguajes proporcionales + stats numéricas. |

## 6. Diseño Visual

### Tema

- **Dark mode** por defecto, con toggle a light mode.
- Persistir preferencia en `localStorage`, respetar `prefers-color-scheme` como valor inicial.

### Paleta de colores

| Token | Dark | Light |
|-------|------|-------|
| Background | `#09090b` (zinc-950) | `#fafafa` (zinc-50) |
| Surface | `#18181b` (zinc-900) | `#ffffff` |
| Text primary | `#fafafa` (zinc-50) | `#09090b` (zinc-950) |
| Text secondary | `#a1a1aa` (zinc-400) | `#71717a` (zinc-500) |
| Accent | `#3b82f6` (blue-500) | `#2563eb` (blue-600) |
| Accent hover | `#60a5fa` (blue-400) | `#1d4ed8` (blue-700) |
| AI gradient | `#8b5cf6` → `#06b6d4` | `#7c3aed` → `#0891b2` |

### Tipografía

- **Headings:** Inter (o Geist Sans) — bold, tracking tight.
- **Body:** Inter — regular, buena legibilidad.
- **Mono:** JetBrains Mono (para stats/código).

### Animaciones (Framer Motion)

| Elemento | Animación |
|---------|-----------|
| Timeline cards | Scroll reveal: fade-in + slide-up (staggered) |
| Page transitions | Layout animation entre `/` y `/projects/[slug]` |
| Hover en cards | Scale sutil (1.02) + shadow elevado |
| Tech badges | Fade-in staggered al entrar la card |
| Activity grid | Fade-in progresivo de izquierda a derecha |
| Screenshots | Zoom suave al abrir lightbox |
| Theme toggle | Rotación del ícono (sol/luna) |

## 7. Deployment

### Vercel

- **Build:** `next build` genera sitio estático (SSG).
- **Output:** todas las páginas pre-renderadas en build time.
- `generateStaticParams` en `/projects/[slug]` lee todos los slugs de `projects/`.

### Dominios

- **raphauy.dev** — dominio principal.
- **rapha.uy** — redirect 301 a raphauy.dev (configurado en Vercel).
- Ambos con HTTPS automático vía Vercel.

### Performance targets

- Lighthouse score: 95+ en todas las categorías.
- LCP < 2.5s, FID < 100ms, CLS < 0.1.
- Bundle size mínimo gracias a SSG + tree shaking.
